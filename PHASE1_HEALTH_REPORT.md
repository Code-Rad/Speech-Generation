### POST /generate

**Purpose:** Generate speech from text. Returns the audio file directly as a binary download.

**Request (JSON body):**
```json
{
  "text": "Breaking news this evening.",
  "profile_id": "anchor_male_en",
  "output_format": "wav",
  "require_cloning": false
}
```

| Field | Type | Required | Default | Notes |
|-------|------|----------|---------|-------|
| `text` | string | Yes | — | 1–5000 characters |
| `profile_id` | string | Yes | — | Must match one of the 6 profile IDs |
| `output_format` | string | No | `"wav"` | `"wav"` or `"mp3"` |
| `require_cloning` | bool | No | `false` | If `true`, skips edge_tts (no voice cloning support) |

**Response (200):** Binary audio file (`audio/wav` or `audio/mpeg`)

**Response headers:**
```
X-Engine-Used: edge_tts
X-Is-Draft: true
X-Duration-Seconds: 3.45
X-Generation-Time: 2.12
X-Voice-Profile: anchor_male_en
X-Language: en
Content-Disposition: attachment; filename="anchor_male_en_20260416_083000_a1b2c3d4.wav"
```

**Error codes:**
- `404` — profile_id not found
- `422` — empty text, invalid output_format, or Pydantic validation error
- `503` — no engine available (e.g., `require_cloning=true` but all cloning engines unavailable)
- `500` — engine succeeded but output file missing (should not occur)

**Phase 1 behaviour:** Always uses `edge_tts` fallback (XTTS unavailable). `X-Is-Draft: true` always.

### POST /clone-voice

**Purpose:** Upload a reference audio file to set up voice cloning for a profile.

**Request (multipart/form-data):**
| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `profile_id` | string (Form) | Yes | Target voice profile |
| `audio_file` | file (File) | Yes | WAV or MP3, 10–60 seconds, ≥ 16 kHz, ≥ 50 KB |
| `generate_sample` | bool (Form) | No (default: true) | Generate a sample clip after upload |

**Audio validation requirements (enforced by AudioValidator):**
- File size: 50 KB – 50 MB
- Extension: `.wav` or `.mp3`
- Duration: 10–60 seconds (warning if < 15s or > 45s)
- Sample rate: ≥ 16,000 Hz (warning if < 22,050 Hz)
- Not pure silence: RMS energy ≥ 0.01

**Processing applied:**
- Stereo → mono (channel average)
- Resample to 22,050 Hz if needed
- Amplitude normalize (peak → 0.95)
- Output: WAV, PCM_16

**Response (200):**
```json
{
  "success": true,
  "profile_id": "anchor_male_en",
  "reference_audio_saved": true,
  "reference_audio_path": "C:\\VoiceForge\\server\\reference_audio\\anchor_male_en_reference.wav",
  "duration_seconds": 22.6,
  "sample_rate": 22050,
  "warnings": [],
  "sample_generated": true,
  "sample_audio_path": "C:\\VoiceForge\\server\\output\\anchor_male_en_sample_20260416_083000_a1b2c3d4.wav",
  "sample_engine_used": "edge_tts",
  "message": "Reference audio uploaded and validated for 'Male News Anchor — English'. Duration: 22.6s at 22050 Hz. Sample generated successfully."
}
```

**Error responses:**
- `404` — profile not found
- `422` — file type wrong, or audio validation failed (body contains `{"detail": {"error": "descriptive message"}}`)
- `500` — file I/O error

**Phase 1 behaviour:** Upload and processing work fully. Sample is generated by edge_tts (ignores reference audio — actual voice cloning activates when XTTS model is loaded).

### POST /generate-batch

**Purpose:** Generate multiple audio segments in a single request.

**Request (JSON body):**
```json
{
  "items": [
    {"text": "First headline.", "profile_id": "anchor_male_en", "output_format": "wav"},
    {"text": "Second headline.", "profile_id": "anchor_female_en", "output_format": "wav"}
  ],
  "require_cloning": false
}
```

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `items` | array | Yes | 1–20 items |
| `require_cloning` | bool | No | Applies to all items |

Each item: `{text: string, profile_id: string, output_format?: string}`

**Response (200):**
```json
{
  "results": [
    {
      "success": true,
      "audio_path": "C:\\VoiceForge\\server\\output\\anchor_male_en_20260416_083000_a1b2c3d4.wav",
      "duration_seconds": 1.8,
      "engine_used": "edge_tts",
      "voice_profile_id": "anchor_male_en",
      "language": "en",
      "is_draft": true,
      "error_message": "",
      "generation_time_seconds": 2.1,
      "download_url": null
    }
  ],
  "total_items": 2,
  "succeeded": 2,
  "failed": 0,
  "total_time_seconds": 4.5
}
```

**Key behaviour:** Failed items have `success: false` and `error_message` populated. Failures do not abort other items. The batch always returns 200 even if some items fail — check `failed` count.
